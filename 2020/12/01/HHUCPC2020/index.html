    A题 ZHAN与武典灵

    纯套路题，区间其实是可以只包含序列的一个值，就遍历找最大值就行了。

    考察点：贪心（数学）

    B题 ZHAN与手机号

    字符串模拟，跟着写就好了。

    注意了，即使是一个标准 111111 位长的纯数字串，也可能不是一个合法的手机号码（例如样例里面的 12812345678​），所以记得在判断函数里面在最后加上对这一类的处理。（要不是样例里面有这么一个，我指不定还得在这题上面画上一大段时间，所以说还是要细心）

    考察点：字符串，模拟

    C题 ZHAN与地摊

    根据贪心，显然应该每次买能买到的最便宜的物品，这样可以使得买的东西最多，且在商品数量最多的前提下，剩下的钱最多。

    那么问题转化为，现在有 nmnmnm 件物品，购买最便宜的 TTT 件物品，使得 TTT 尽量大。

    如果使用暴力，那么我们应该将这 nmnmnm 件物品排序，然后扫一遍，但是时间复杂度为 O((nm)log⁡nm)O((nm)\log {nm})O((nm)lognm)，铁 TLETLETLE，而且空间复杂度也接受不了。

    考虑到这 nmnmnm 件物品的价格由 nnn 件商品和 mmm 种优惠构造而成，那么我们可以先对这两个序列进行排序，然后将 nnn 个物品使用最大优惠之后扔进一个堆里面，之后每次选出堆顶元素，买下这个物品，然后将该件物品使用次一档优惠继续扔进堆里面，这样便可以保证每次买的物品都是当下可以买到的最便宜的物品。依次循环，直到钱不够为止。复杂度为 O(nlog⁡n)O(n\log{n})O(nlogn)，可以过。

    有一个小细节，当优惠用完之后，每次直接将物品按照原价扔进堆里面，忘了的话要么 WA（没有继续扔），要么 ​RE​（不停用下一个优惠但是没算好用没用完，然后越界了）。

    考察点：贪心，堆

    D题 ZHAN与人工智能

    纯粹图上遍历罢了，DFS 或者 BFS​ 都可以。（并查集维护图联通似乎也可以）

    细节有点多，要多审题，而且最好先加上debug语句调试一下，确定没问题之后再交上去。（调bug都调吐了）

    考察点：图上遍历/并查集

    E题 ZHAN与形政论文

    最短路板子罢了，数据范围这么小，不管邻接矩阵还是邻接表，vector 维护还是链式前向星，Dijkstra 用不用堆优化，卡不卡 SPFA，都能轻松 AC 这题。如果常数小，你甚至可以直接上 Floyd 水过去。

    看到这个一开始很开心，结果 NM 把无向图当有向图写，结果被罚时一波，很难受…

    考点：最短路

    F题 ZHAN与开封菜

    二维完全背包，有两个维度（重量和价格），使得重量控制在一定维度，价格不超的情况下能够使得满意度最大 。

    我们先给出 DP 方程：

    dp(i,j)=max⁡1≤k≤n{dp(i−w[k],j−v[k])+t[k]}dp(i,j)=\max\limits_{1\leq k \leq n}{\{dp(i-w[k],j-v[k])+t[k]\}} dp(i,j)=1≤k≤nmax​{dp(i−w[k],j−v[k])+t[k]}

    边界为dp(0,0)=0dp(0,0)=0dp(0,0)=0。

    这个转移有点离谱，所以额外给一个 visvisvis 数组来记录下这个状态是否可行，避免出现转移错误的情况。（这里给zsz大佬惋惜一下，可能就是被这题整麻了）

    DP完了之后，在指定区间统计答案，便有

    ans=max⁡a≤i≤b,0≤j≤c,vis[i][j]=1dp[i][j]ans=\max\limits_{a\leq i \leq b,0\leq j \leq c,vis[i][j]=1}{dp[i][j]} ans=a≤i≤b,0≤j≤c,vis[i][j]=1max​dp[i][j]

    考点：完全背包，多维背包

    J题 ZHAN与防AK题

    构造一个序列，序列中任意相邻三项不相等，尝试使其在和为 nnn 的基础上，使得序列长度最长。

    显然，我们一下子就想到了 1,2,31,2,31,2,3 序列。显然，当 n≥6n \geq 6n≥6 时，总像这样做题是没问题的。

    但下面就尴尬了，考场上把我想了那么长时间都 WA（那么多人都 AC 了，把我急得）。

    最后发现，要求和 **恰好为 n ，而不是大于等于 n **。

    这样就不能无脑 123 了，需要对 1 到 5 分别处理。

    显然，1=1,2=2,3=1+2,4=1+3,5=2+31=1,2=2,3=1+2,4=1+3,5=2+31=1,2=2,3=1+2,4=1+3,5=2+3，这样使得恰好满足的情况下天数最大，这样就能 AC 了，芜湖。（考场上面差点没有过，吓死我了）

    考察点：数学规律

    H题/I题 ZHAN与黄焖鸡(普通版/增强版)

    暴力的复杂度是 O(n2m)O(n^2m)O(n2m) ，铁定 TLE 。

    注意到题目里面有“最小值最大”的说法，而且发现答案满足单调性，显然可以二分答案。

    发现两组题目的唯一差别在于 mmm 的大小（普通版为 10，增强版为 15），结合二分本身的复杂度，估计判定的复杂度大约为 O(2m)O(2^m)O(2m)一级。

    突然发现，以二分值 xxx 为界，数列中大于等于 xxx 者为 1，小于 xxx 者为 0，这样就可以转化成一个 01 串（有状压 DP 那味了），两个序列合并之后满足条件的充要条件为 两个 01 串 做 ororor 运算之后全部为 1。

    可惜的是，如果这样写，每次判定的复杂度是 O(nm+n2)O(nm+n^2)O(nm+n2)，照样T飞。

    对 H 题：

    突然发现（指看了题解），m≤10m \leq 10m≤10，01 串最多只有 2m2^m2m 种,那么可以用 O(nm)O(nm)O(nm) 的时间处理下每个数列的 01 串，然后枚举的时候就变成 O(22m)O(2^{2m})O(22m)了，这样总的判定的复杂度便成了 O(nm+22m)O(nm+2^{2m})O(nm+22m)，总的复杂度为 O((nm+22m)log⁡109)O((nm+2^{2m})\log{10^9})O((nm+22m)log109)，压一压常数就可以过了。

    对了，在这里补充一个小知识：如果想在两个数的每一位进行 andandand 或者 ororor 运算，用 &\&& 和 ∣|∣ 而不是 &&\&\&&& 和 ∣∣||∣∣。后者仅在逻辑正确与否上面成立（0 和 1），但是数学上不等价。换言之，在 ififif 语句中用后者刚刚好，但如果是做数学上的逻辑运算，还是用前者。（早上因为这个调了半天就离谱）
